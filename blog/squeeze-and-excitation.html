Squeeze-and-Excitation networks

Boost your CNN accuracy

Posted by Taha Samavati on Spetember 24, 2021

Languages: English فارسی

Back in 2018, with the introduction of the SE (Squeeze and Excitation) block, the ImageNet accuracy improved from last year by 2.5%. This block takes into account the interdependencies between different feature channels of CNNs. In this post we're going to see "Why?" and "How?" these blocks can improve the performance of any convolutional neural network with little computational cost.

Main Idea

Consider a convolutional layer in a CNN, the weights of each channel are equal when computing the output. In other words, the weights are a 2D matrix that are convolved with every channel. The SE block, However, weights each channel adaptively based on the context of each channel. To put it more simply the SE block takes into account how relevant each feature map is when computing output.

The Details

Now let's see a more detailed explanation of how the SE block's output is produced. At first, the block gains a global understanding of each channel by squeezing each one to a single numeric value. After This step, we have a vector of size n (determined by the number of channels in the input). This vector is then fed to a two-layer feed-forward network. The network outputs a vector of the same size (n). These n values are the weights of each channel that are used to scale each channel of the input tensor based on its context.

The Implementation

The SE block is defined below as a function. The function takes the feature map and number of channels as input. GlobalAveragePooling converts each channel to a single numerical value (Squeezing part). In the following a stack of two Dense blocks transform the n values to n weights for each channel (Excitation). Finally, the output is computed by applying weights to the channels by multiplication.

def se_block(in_block, ch, ratio=16):
    y = GlobalAveragePooling2D()(in_block)
    y = Dense(ch//ratio, activation='relu')(y)
    y = Dense(ch, activation='sigmoid')(y)
    return multiply()([in_block, y])

The first layer of the feed-forward network has a relu activation function and the second (last) layer has a sigmoid activation function acting as a smooth gating function.

Pytorch Implementation

class SE_Block(nn.Module):   def __init__(self, c, r=16):
    super().__init__()
    self.squeeze = nn.AdaptiveAvgPool2d(1)
    self.excitation = nn.Sequential(
        nn.Linear(c, c // r, bias=False),
        nn.ReLU(inplace=True),
        nn.Linear(c // r, c, bias=False),
        nn.Sigmoid()
    )

  def forward(self, x):
    bs, c, _, _ = x.shape
    y = self.squeeze(x).view(bs, c)
    y = self.excitation(y).view(bs, c, 1, 1)
    return x * y.expand_as(x)
